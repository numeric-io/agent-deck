**Agent-deck SKILL.md frontmatter fix:**
Change the YAML frontmatter from:
```yaml
---
name: agent-deck
description: Terminal session manager for AI coding agents. Use when user mentions...

---
```
To:
```yaml
---
name: agent-deck
description: Terminal session manager for AI coding agents. Use when user mentions...

  compatibility: "claude, opencode"
---
```

Only the frontmatter changes. The rest of the file (everything after the closing `---`) stays exactly as-is.

**Session-share SKILL.md frontmatter fix:**
Same change: move `compatibility: claude, opencode` from top-level to `metadata:` map with the value quoted.

**Session-share path resolution addition:**
Add a `## Script Path Resolution (IMPORTANT)` section immediately after the first heading and version line (after `**Version:** 1.0 | **Privacy:**...`), before `## Quick Start`. Use this exact content:

```markdown
## Script Path Resolution (IMPORTANT)

This skill includes helper scripts in its `scripts/` subdirectory. When Claude Code loads this skill, it shows a line like:

```
Base directory for this skill: /path/to/.../skills/session-share
```

**You MUST use that base directory path to resolve all script references.** Store it as `SKILL_DIR`:

```bash
# Set SKILL_DIR to the base directory shown when this skill was loaded
SKILL_DIR="/path/shown/in/base-directory-line"

# Then run scripts as:
$SKILL_DIR/scripts/export.sh
$SKILL_DIR/scripts/import.sh ~/Downloads/session-file.json
```
```

Then update ALL bare `scripts/` references throughout the file to use `$SKILL_DIR/scripts/` instead. Specifically:
- Quick Start section: `scripts/export.sh` becomes `$SKILL_DIR/scripts/export.sh`, `scripts/import.sh` becomes `$SKILL_DIR/scripts/import.sh`
- Export Session section: `scripts/export.sh [options]` and all examples
- Import Session section: `scripts/import.sh <file-path> [options]` and all examples
- Workflow section: same pattern

Do NOT change the `scripts/` references that appear inside the Export File Format JSON block or the Technical Details section (those are describing file paths, not commands to run).
