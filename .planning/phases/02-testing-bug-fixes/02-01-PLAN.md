  <name>Task 1: Status transition cycle tests (TEST-01)</name>
  <files>internal/session/status_lifecycle_test.go</files>
  <behavior>
    - Test: Shell session starts as StatusStarting (when command provided), transitions to StatusIdle after grace period + UpdateStatus()
    - Test: Shell session without command starts as StatusIdle (no StatusStarting)
    - Test: After Kill(), session status is StatusError and tmux session no longer exists
    - Test: UpdateStatus() returns StatusError when tmux session is gone (session killed externally)
    - Test: Hook fast path updates: setting hookStatus="running" on Claude-compatible instance makes UpdateStatus() return StatusRunning
    - Test: Hook fast path waiting with acknowledged=true transitions to StatusIdle (not StatusWaiting)
    - Test: Shell tool sessions ignore hook fast path entirely, use tmux polling
  </behavior>
  <action>
Create `internal/session/status_lifecycle_test.go` in the `session` package.

Use `skipIfNoTmuxServer(t)` for all integration tests that create real tmux sessions. Use `defer func() { _ = inst.Kill() }()` for cleanup.

**Integration tests (require tmux):**

1. `TestStatusCycle_ShellSessionWithCommand`: Create instance with `Tool: "shell"`, `Command: "sleep 30"`. Call `Start()`. Assert status is `StatusStarting`. Wait 2 seconds, call `UpdateStatus()`. Assert status is NOT `StatusStarting` (should be running or idle depending on tmux detection). Call `Kill()`. Assert `StatusError`. Assert `inst.Exists() == false`.

2. `TestStatusCycle_ShellSessionNoCommand`: Create instance with `Tool: "shell"`, no command. Call `Start()`. Assert status remains `StatusIdle` (no command means no StatusStarting). Wait 2 seconds, call `UpdateStatus()`. Assert `StatusIdle`.

3. `TestStatusCycle_KilledExternally`: Create instance, start it. Use `exec.Command("tmux", "kill-session", "-t", inst.GetTmuxSession().Name())` to kill externally. Call `UpdateStatus()`. Assert `StatusError`.

**Unit tests (no tmux needed):**

4. `TestHookFastPath_RunningStatus`: Create instance with `Tool: "claude"`. Manually set `hookStatus` and `hookLastUpdate` fields (they are unexported, so use `UpdateHookStatus(&HookStatus{Status: "running", ...})` method). Call `UpdateStatus()` and verify status is `StatusRunning`. NOTE: This test will need tmux since UpdateStatus checks tmux existence first. If tmux is unavailable, verify through the hook_watcher pattern instead (set up a mock-like scenario using `StatusFileWatcher` pattern from `hook_watcher_test.go`).

5. `TestHookFastPath_WaitingAcknowledged`: Table-driven test for hook waiting status. When acknowledged=true, status should be `StatusIdle`. When acknowledged=false, status should be `StatusWaiting`. This tests the branch at lines 2336-2340 of instance.go.

Use `require` from testify for fatal assertions, `assert` for non-fatal checks. Follow the table-driven test pattern from the research document.

**Important timing note:** After `inst.Start()`, always wait at least 2 seconds before calling `UpdateStatus()` to clear the 1.5-second grace period (line 2263 of instance.go). The grace period prevents status detection during tmux initialization.
